# nanobot 使用手册

本文档为 nanobot 的使用指南，带你从零开始掌握这个超轻量级 AI 助手。

---

## 1. 安装与初始化

### 1.1 前置要求

- **Bun** ≥ 1.0（必须）
- 如果需要飞书集成：飞书企业管理员账号

安装 Bun：
```bash
# macOS / Linux
curl -fsSL https://bun.sh/install | bash

# Windows (需要 WSL 或 Docker)
# 建议使用 WSL2 + Bun
```

### 1.2 安装 nanobot

```bash
# 克隆项目
git clone https://github.com/HKUDS/nanobot.git
cd nanobot

# 安装依赖
bun install

# 编译 TypeScript（可选，首次运行会自动编译）
bun run build
```

### 1.3 初始化配置

```bash
# 方式一：使用 bun 运行
bun run src/cli/commands.ts onboard

# 方式二：使用编译后的二进制
./bin/nanobot onboard
```

初始化过程会：
1. 创建配置文件 `~/.nanobot/config.json`
2. 创建工作空间目录 `~/.nanobot/`
3. 创建引导文件：
   - `AGENTS.md` - Agent 指令
   - `SOUL.md` - 人格设定
   - `USER.md` - 用户信息
   - `HEARTBEAT.md` - 心跳任务清单
   - `memory/MEMORY.md` - 长期记忆

## 3. 基本用法

### 3.1 单次对话

```bash
# 发送一条消息，得到回复后退出
bun run src/cli/commands.ts agent -m "你好"

# 或使用二进制
./bin/nanobot agent -m "你好"
```

### 3.2 交互模式

```bash
# 启动交互式对话（类似 ChatGPT CLI）
bun run src/cli/commands.ts agent

# 退出：输入 quit、exit、q 或按 Ctrl+C
```

交互模式下支持的操作：
- 输入内容并回车发送
- `quit` / `exit` / `q` - 退出
- `clear` - 清空当前会话历史
- `help` - 查看帮助

### 3.3 指定模型

```bash
# 使用指定模型
bun run src/cli/commands.ts agent -m "你好" --model gpt-4o

# 交互模式指定模型
bun run src/cli/commands.ts agent --model gpt-4o
```

## 4. 飞书集成

### 4.1 创建飞书应用

1. 访问 https://open.feishu.cn/
2. 创建企业自建应用
3. 获取 `App ID` 和 `App Secret`

### 4.2 配置应用权限

在飞书开放平台的应用管理中，添加以下权限：
- `im:message:send_as_bot` - 发送消息
- `im:message:receive` - 接收消息
- `im:resource:download` - 下载图片

### 4.3 配置 Webhook

1. 创建版本并发布
2. 添加重定向 URL（需要 HTTPS 或使用内网穿透）
3. 获取 Webhook 地址

### 4.4 配置 nanobot

编辑 `~/.nanobot/config.json`：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "app_id": "你的AppID",
      "app_secret": "你的AppSecret",
      "allowFrom": []
    }
  }
}
```

- `allowFrom`：允许发消息的用户 ID 列表，空数组表示允许所有人
- 可选配置：
  - `verifyToken`：验证回调请求的 token
  - `encryptKey`：回调消息解密密钥

### 4.5 启动网关服务

```bash
# 启动网关服务（需要保持运行）
bun run src/cli/commands.ts gateway

# 或使用二进制
./bin/nanobot gateway
```

启动后，飞书发送的消息会自动触发 Agent 处理。

## 5. 定时任务 (Cron)

### 5.1 添加定时任务

```bash
# 方式一：使用 cron 表达式（每天 9:00 执行）
bun run src/cli/commands.ts cron add \
  --name "morning-greeting" \
  --message "早安！新的一天开始了" \
  --cron "0 9 * * *"

# 方式二：使用固定间隔（每小时执行一次）
bun run src/cli/commands.ts cron add \
  --name "hourly-check" \
  --message "检查系统状态" \
  --every 3600

# 方式三：指定具体时间（一次性任务）
bun run src/cli/commands.ts cron add \
  --name "one-time-task" \
  --message "完成任务" \
  --at "2026-01-01 00:00:00"
```

### 5.2 查看任务列表

```bash
bun run src/cli/commands.ts cron list
```

输出示例：
```
名称              调度方式         下次执行时间
morning-greeting cron: 0 9 * * *  2026-01-26 09:00:00
hourly-check     every: 3600s    2026-01-26 01:00:00
```

### 5.3 删除定时任务

```bash
bun run src/cli/commands.ts cron remove --name "morning-greeting"
```

### 5.4 Cron 表达式说明

| 字段 | 值 | 示例 |
|------|-----|------|
| 分钟 | 0-59 | `0` - 每小时整点 |
| 小时 | 0-23 | `9` - 早上 9 点 |
| 日期 | 1-31 | `*` - 每天 |
| 月份 | 1-12 | `*` - 每月 |
| 星期 | 0-6 | `1-5` - 工作日 |

常用表达式：
- `0 9 * * *` - 每天早上 9 点
- `0 9 * * 1-5` - 工作日早上 9 点
- `*/30 * * * *` - 每 30 分钟
- `0 * * * *` - 每小时

## 6. 心跳服务 (Heartbeat)

### 6.1 什么是心跳服务

心跳服务会定期检查 `HEARTBEAT.md` 文件，如果文件中有待办任务，就自动执行。

### 6.2 配置心跳服务

编辑 `~/.nanobot/config.json`：

```json
{
  "heartbeat": {
    "enabled": true,
    "intervalSeconds": 1800
  }
}
```

- `enabled`：是否启用心跳服务
- `intervalSeconds`：检查间隔，默认 1800 秒（30 分钟）

### 6.3 编写任务清单

编辑 `~/.nanobot/HEARTBEAT.md`：

```markdown
# 待办任务

- [ ] 检查服务器状态并报告
- [ ] 发送每日简报
```

当心跳服务检测到未完成的任务时，会让 Agent 执行任务。

### 6.4 完成任务

任务执行完成后，在任务前加 `x`：

```markdown
# 待办任务

- [x] 检查服务器状态并报告
- [ ] 发送每日简报
```

## 8. 技能 (Skills)

### 8.1 什么是技能

技能是预先封装好的工具集合，可以扩展 Agent 的能力。

### 8.2 内置技能

项目内置了以下技能（位于 `src/skills/`）：

| 技能名称 | 功能 |
|----------|------|
| `git-commit` | Git 提交辅助 |
| `skill-manager` | 技能管理 |
| `tavily` | 搜索 API |
| `business-analyzer` | 业务分析 |

### 8.3 安装自定义技能

将技能文件夹放入 `~/.nanobot/skills/` 目录。

技能结构：
```
~/.nanobot/skills/my-skill/
├── skill.yaml          # 技能配置
├── prompts/           # 提示词
│   └── system.md
└── tools/             # 工具定义
    └── tools.yaml
```

### 8.4 配置 Always-Skills

编辑 `~/.nanobot/config.json`：

```json
{
  "skills": {
    "always": ["git-commit", "tavily"]
  }
}
```

`always` 列表中的技能会在每次对话时自动加载。

## 9. 配置文件详解

### 9.1 完整配置项

```json
{
  "agents": {
    "defaults": {
      "model": "claude-sonnet-4-20250514",
      "max_tool_iterations": 100,
      "max_tokens": 4096,
      "temperature": 0.7
    }
  },
  "providers": {
    "anthropic": {
      "apiKey": "sk-ant-xxx",
      "apiBase": "https://api.anthropic.com"
    },
    "openai": {
      "apiKey": "sk-proj-xxx",
      "apiBase": "https://api.openai.com/v1"
    }
  },
  "channels": {
    "feishu": {
      "enabled": true,
      "app_id": "xxx",
      "app_secret": "xxx",
      "allowFrom": [],
      "verifyToken": "",
      "encryptKey": ""
    }
  },
  "cron": {
    "enabled": true
  },
  "heartbeat": {
    "enabled": true,
    "intervalSeconds": 1800
  },
  "skills": {
    "always": []
  }
}
```

### 9.2 配置项说明

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `agents.defaults.model` | string | 默认使用的模型 |
| `agents.defaults.max_tool_iterations` | number | 单次对话最大工具调用次数 |
| `agents.defaults.max_tokens` | number | 单次响应最大 token 数 |
| `agents.defaults.temperature` | number | 生成温度 (0-2) |
| `providers.*.apiKey` | string | API Key |
| `providers.*.apiBase` | string | API 基础地址 |
| `channels.feishu.enabled` | boolean | 是否启用飞书 |
| `channels.feishu.allowFrom` | string[] | 允许的用户 ID，空数组表示所有人 |
| `cron.enabled` | boolean | 是否启用定时任务 |
| `heartbeat.enabled` | boolean | 是否启用心跳服务 |
| `heartbeat.intervalSeconds` | number | 心跳间隔（秒） |
| `skills.always` | string[] | 自动加载的技能列表 |

## 11. 常见问题

### 11.1 安装问题

**Q: bun: command not found**
A: 确保 Bun 已正确安装并添加到 PATH。运行 `source ~/.bashrc` 或重启终端。

**Q: 编译报错**
A: 确保使用 `bun install` 安装依赖，而不是 npm/yarn/pnpm。

### 11.2 运行问题

**Q: 提示 "No API key configured"**
A: 检查 `~/.nanobot/config.json` 是否正确配置了 API Key。

**Q: 飞书消息收不到**
A: 
1. 确认网关服务已启动：`bun run src/cli/commands.ts gateway`
2. 检查飞书应用的回调 URL 是否正确
3. 确认应用已发布上线

**Q: 工具执行失败**
A: 查看终端输出的错误信息，常见原因：
- 权限不足（文件/目录权限）
- 命令不存在（检查 PATH）
- 网络问题（代理配置）

### 11.3 使用问题

**Q: 对话历史太长**
A: 使用 `clear_context` 工具清理上下文，或配置 `ContextCleaner` 自动清理。

**Q: 模型响应太慢**
A: 
- 检查网络连接
- 尝试使用更快的模型（如 claude-3-haiku）
- 减少 max_tokens 限制

## 12. 快捷命令汇总

```bash
# 开发/构建
bun install          # 安装依赖
bun run build       # 编译 TypeScript
bun test            # 运行测试

# 运行
bun run src/cli/commands.ts agent -m "消息"  # 单次对话
bun run src/cli/commands.ts agent             # 交互模式
bun run src/cli/commands.ts gateway           # 启动网关

# 定时任务
bun run src/cli/commands.ts cron add --name "名称" --message "消息" --cron "0 9 * * *"
bun run src/cli/commands.ts cron list
bun run src/cli/commands.ts cron remove --name "名称"

# 技能
bun run src/cli/commands.ts skill list
bun run src/cli/commands.ts skill add --path /path/to/skill

# Makefile 快捷方式
make install   # bun install
make build    # bun run build
make test     # bun test
make run MSG='Hello'
make agent    # 交互模式
make gateway  # 网关服务
make cron-list
```

### 13.3 图片理解

飞书发送图片时，nanobot 会自动：
1. 下载图片
2. 转换为 base64
3. 发送给 Vision 模型分析
4. 返回图片描述

确保使用支持 Vision 的模型（如 Claude 3.5 Sonnet、GPT-4o）。
